## Перегрузка операторов

### Введение: зачем перегружать операторы?

Представьте, что мы создали класс для работы с комплексными числами:

```cpp
class Complex {
public:
    double real;
    double imag;

    Complex(double r = 0, double i = 0) : real(r), imag(i) {}
};
```

Без перегрузки операторов сложение двух комплексных чисел пришлось бы делать через обычный метод:

```cpp
Complex add(const Complex& other) const {
    return Complex(real + other.real, imag + other.imag);
}

// Использование:
Complex c1(1, 2), c2(3, 4);
Complex c3 = c1.add(c2);   // Неудобно и неинтуитивно
```

Гораздо естественнее было бы писать `Complex c3 = c1 + c2;`. Именно это позволяет сделать перегрузка операторов. Она **не создаёт новых операторов** и **не изменяет приоритет или ассоциативность** существующих, а лишь определяет их поведение для пользовательских типов.

**Важно:** Перегрузка операторов должна сохранять интуитивное поведение. Не стоит делать оператор `+` вычитанием, а оператор `<<` — чтением из файла. Это нарушит ожидания читающего код и приведёт к ошибкам.

---

### 1. Основы синтаксиса

Перегрузить оператор можно двумя способами:

1. **Как метод класса** (для операторов, где левый операнд — объект класса).
2. **Как свободную (обычную) функцию** (часто с ключевым словом `friend`, если нужен доступ к приватным членам).

Синтаксис перегрузки: `operator<знак_оператора>`.

Пример для оператора `+`:

```cpp
// Как метод класса
class Complex {
public:
    Complex operator+(const Complex& other) const {
        return Complex(real + other.real, imag + other.imag);
    }
};

// Как свободная функция
Complex operator+(const Complex& lhs, const Complex& rhs) {
    return Complex(lhs.real + rhs.real, lhs.imag + rhs.imag);
}
```

#### Какой способ выбрать?

- **Метод класса** обязателен для операторов, которые должны изменять сам объект (`=`, `+=`, `[]` для неконстантных объектов) или требуют доступа к `this` (`->`, `()`).
- **Свободная функция** предпочтительна для бинарных операторов, где левый операнд может быть не объектом класса (например, число `5 + complex`). Также свободная функция обеспечивает симметричность неявных преобразований.

**Правило большого пальца:** Если оператор не должен менять левый операнд и левый операнд может быть другого типа — делайте свободную функцию.

### 2. Перегрузка арифметических операторов

Продолжим с классом `Complex`. Реализуем основные арифметические операторы.

#### Операторы `+`, `-`, `*`, `/` (бинарные)

Рекомендуется реализовывать их как свободные функции, чтобы поддерживать выражения вида `double + Complex`.

```cpp
#include <iostream>

class Complex {
private:
    double re, im;
public:
    Complex(double r = 0, double i = 0) : re(r), im(i) {}

    // Методы доступа (для свободных функций)
    double real() const { return re; }
    double imag() const { return im; }

    // Составные операторы присваивания (+=, -=, *=, /=) реализуем внутри класса
    Complex& operator+=(const Complex& other) {
        re += other.re;
        im += other.im;
        return *this;
    }

    Complex& operator-=(const Complex& other) {
        re -= other.re;
        im -= other.im;
        return *this;
    }
};

// Бинарные операторы через свободные функции, используя += для эффективности
Complex operator+(const Complex& lhs, const Complex& rhs) {
    Complex result = lhs;
    result += rhs;
    return result;
}

Complex operator-(const Complex& lhs, const Complex& rhs) {
    Complex result = lhs;
    result -= rhs;
    return result;
}

// Оператор вывода (объявлен как друг, но можно использовать геттеры)
std::ostream& operator<<(std::ostream& os, const Complex& c) {
    os << c.real() << " + " << c.imag() << "i";
    return os;
}

int main() {
    Complex c1(3, 2), c2(1, 7);
    Complex c3 = c1 + c2;      // 4 + 9i
    Complex c4 = c1 - c2;      // 2 - 5i
    std::cout << c3 << "\n";
}
```

**Пояснения:**
- Составные операторы (`+=`, `-=`) реализованы как методы (меняют состояние объекта) и возвращают ссылку на себя для цепочек (`a += b += c`).
- Обычные арифметические операторы (`+`, `-`) реализованы через составные: это уменьшает дублирование кода и гарантирует согласованность.
- Возврат по значению из `+` и `-` обязателен, так как создаётся новый объект.

#### Унарные операторы `+` и `-`

```cpp
// Унарный минус
Complex operator-(const Complex& c) {
    return Complex(-c.real(), -c.imag());
}

// Унарный плюс (обычно ничего не делает, но может использоваться для симметрии)
Complex operator+(const Complex& c) {
    return c;
}
```

### 3. Перегрузка операторов сравнения

Для того чтобы можно было сравнивать объекты класса (например, для сортировки или использования в ассоциативных контейнерах), необходимо перегрузить операторы `==`, `!=`, `<`, `<=`, `>`, `>=`.

В C++20 появился оператор "космический корабль" `<=>`, но пока рассмотрим традиционный способ (до C++20).

```cpp
class Complex {
    // ...
public:
    bool operator==(const Complex& other) const {
        return re == other.re && im == other.im;
    }

    bool operator!=(const Complex& other) const {
        return !(*this == other);   // Используем уже реализованный ==
    }

    // Для сортировки комплексные числа обычно сравнивают по модулю, 
    // но для примера сравним по действительной части, потом по мнимой.
    bool operator<(const Complex& other) const {
        if (re != other.re) return re < other.re;
        return im < other.im;
    }

    // Остальные операторы можно выразить через < и ==
    bool operator<=(const Complex& other) const {
        return !(other < *this);
    }
    bool operator>(const Complex& other) const {
        return other < *this;
    }
    bool operator>=(const Complex& other) const {
        return !(*this < other);
    }
};
```

**Примечание:** Достаточно реализовать `==` и `<`, остальные можно выразить через них. В C++20 достаточно реализовать `operator<=>` и `operator==`, а остальные компилятор сгенерирует автоматически.

### 4. Перегрузка операторов ввода/вывода

Операторы `<<` и `>>` для работы с потоками **обязательно** перегружаются как свободные функции, так как левым операндом является объект потока (`std::ostream&` или `std::istream&`), а не ваш класс.

```cpp
#include <iostream>

class Point {
private:
    int x, y;
public:
    Point(int x = 0, int y = 0) : x(x), y(y) {}

    // Оператор вывода
    friend std::ostream& operator<<(std::ostream& os, const Point& p) {
        os << "(" << p.x << ", " << p.y << ")";
        return os;
    }

    // Оператор ввода
    friend std::istream& operator>>(std::istream& is, Point& p) {
        // Ожидаем формат: x y  или (x, y)
        char ch;
        is >> ch;
        if (ch == '(') {
            is >> p.x >> ch >> p.y >> ch; // читаем ')' в ch
        } else {
            is.putback(ch);
            is >> p.x >> p.y;
        }
        return is;
    }
};

int main() {
    Point p1(10, 20);
    std::cout << "Point: " << p1 << "\n";

    Point p2;
    std::cout << "Enter point (format: x y or (x,y)): ";
    std::cin >> p2;
    std::cout << "You entered: " << p2 << "\n";
}
```

**Важно:**
- Операторы возвращают ссылку на поток, чтобы можно было составлять цепочки: `cout << a << b`.
- Оператор ввода должен проверять корректность ввода и при необходимости устанавливать флаги ошибок потока.

### 5. Перегрузка оператора присваивания `=`

Оператор присваивания **должен быть методом класса**. Он определяет, как объекту присваивается значение другого объекта того же типа. Если вы его не перегрузите, компилятор сгенерирует **поверхностное копирование** (побитовое для тривиальных типов). Для классов, управляющих ресурсами (память, файлы), нужна собственная реализация.

```cpp
class String {
private:
    char* data;
    size_t len;

    void copyFrom(const String& other) {
        len = other.len;
        data = new char[len + 1];
        std::copy(other.data, other.data + len + 1, data);
    }

    void free() {
        delete[] data;
    }

public:
    String(const char* s = "") {
        len = std::strlen(s);
        data = new char[len + 1];
        std::copy(s, s + len + 1, data);
    }

    // Конструктор копирования
    String(const String& other) {
        copyFrom(other);
    }

    // Оператор присваивания (копирующее присваивание)
    String& operator=(const String& other) {
        if (this != &other) {           // Защита от самоприсваивания
            free();
            copyFrom(other);
        }
        return *this;
    }

    ~String() { free(); }
};
```

**Ключевые моменты:**
- Проверка на самоприсваивание (`if (this != &other)`).
- Освобождение старых ресурсов перед копированием новых.
- Возврат ссылки на себя (`return *this`) для цепочек `a = b = c`.
- Копирующее присваивание тесно связано с конструктором копирования и деструктором (правило трёх: если нужен один из них, скорее всего нужны все три).

В C++11 появилось **перемещающее присваивание** (`operator=(T&&)`), но это отдельная тема (move-семантика).

### 6. Перегрузка оператора индексации `[]`

Используется для доступа к элементам по индексу (как в массивах). Обычно перегружают две версии: константную и неконстантную.

```cpp
class Array {
private:
    int* data;
    size_t size;
public:
    Array(size_t n) : size(n), data(new int[n]()) {}
    ~Array() { delete[] data; }

    // Неконстантная версия: позволяет изменять элементы
    int& operator[](size_t index) {
        return data[index];   // В реальном коде добавить проверку границ
    }

    // Константная версия: только чтение для константных объектов
    const int& operator[](size_t index) const {
        return data[index];
    }
};

int main() {
    Array arr(5);
    arr[2] = 42;                    // Используется неконстантная версия
    const Array& carr = arr;
    std::cout << carr[2] << "\n";   // Используется константная версия
    // carr[2] = 10; // Ошибка компиляции: carr[2] возвращает const int&
}
```

### 7. Перегрузка операторов инкремента и декремента (`++`, `--`)

Существуют префиксная (`++x`) и постфиксная (`x++`) формы. Они различаются сигнатурой: постфиксная принимает фиктивный параметр типа `int`.

```cpp
class Counter {
    int value;
public:
    Counter(int v = 0) : value(v) {}

    // Префиксный ++ (возвращает ссылку на изменённый объект)
    Counter& operator++() {
        ++value;
        return *this;
    }

    // Постфиксный ++ (возвращает копию до изменения)
    Counter operator++(int) {
        Counter old = *this;
        ++value;      // или value++, но обычно вызывают префиксный для единообразия
        return old;
    }

    int get() const { return value; }
};

int main() {
    Counter c(5);
    std::cout << (++c).get() << "\n"; // 6, c теперь 6
    std::cout << (c++).get() << "\n"; // 6 (возвращает старое значение), c теперь 7
    std::cout << c.get() << "\n";     // 7
}
```

**Важно:** Префиксный оператор более эффективен (не создаёт временную копию). Рекомендуется реализовывать постфиксный через префиксный.

### 8. Перегрузка оператора вызова функции `()`

Позволяет объекту вести себя как функция (функтор). Часто используется в алгоритмах STL.

```cpp
class Multiplier {
    int factor;
public:
    Multiplier(int f) : factor(f) {}
    int operator()(int x) const {
        return x * factor;
    }
};

int main() {
    Multiplier times5(5);
    std::cout << times5(10) << "\n"; // 50

    std::vector<int> v = {1,2,3,4};
    std::transform(v.begin(), v.end(), v.begin(), Multiplier(3));
    // v = {3,6,9,12}
}
```

### 9. Перегрузка оператора приведения типа

Позволяет неявно или явно преобразовывать объект в другой тип.

```cpp
class Fraction {
    int num, den;
public:
    Fraction(int n, int d) : num(n), den(d) {}
    // Явное приведение к double (C++11 и выше)
    explicit operator double() const {
        return static_cast<double>(num) / den;
    }
};

int main() {
    Fraction f(3, 4);
    // double d = f; // Ошибка: explicit запрещает неявное преобразование
    double d = static_cast<double>(f); // OK
    std::cout << d << "\n"; // 0.75
}
```

Без `explicit` преобразование было бы неявным, что может привести к неожиданным ошибкам.

### 10. Перегрузка операторов работы с памятью `new` и `delete`

Можно перегрузить глобально или для конкретного класса. Используется для кастомного управления памятью (например, для пулов объектов). Для новичков это редко нужно, но важно знать о существовании такой возможности.

```cpp
class MyClass {
public:
    void* operator new(size_t size) {
        std::cout << "Custom new for MyClass, size " << size << "\n";
        return ::operator new(size);
    }
    void operator delete(void* ptr) {
        std::cout << "Custom delete for MyClass\n";
        ::operator delete(ptr);
    }
};
```

### 11. Важные правила и ограничения

- **Нельзя перегрузить** операторы: `::` (разрешение области видимости), `.` (доступ к члену), `.*` (доступ к члену через указатель), `?:` (тернарный), `sizeof`, `typeid`, `alignof`, `noexcept`.
- **Нельзя создать новый оператор** (например, `**` для возведения в степень).
- **Нельзя изменить арность** (количество операндов) оператора. Например, унарный `+` остаётся унарным, бинарный `-` — бинарным.
- **Нельзя изменить приоритет и ассоциативность**.
- **Хотя бы один операнд должен быть пользовательского типа** (нельзя перегрузить `int + int`).

### 12. Практический пример: класс `Rational` (рациональное число)

Соберём всё вместе в комплексном примере класса для работы с дробями, демонстрирующем перегрузку большинства операторов.

```cpp
#include <iostream>
#include <numeric> // для gcd

class Rational {
private:
    int num;   // числитель
    int den;   // знаменатель (всегда > 0)

    void reduce() {
        int g = std::gcd(num, den);
        num /= g;
        den /= g;
        if (den < 0) {   // знаменатель делаем положительным
            den = -den;
            num = -num;
        }
    }

public:
    Rational(int n = 0, int d = 1) : num(n), den(d) {
        if (den == 0) throw std::invalid_argument("Denominator cannot be zero");
        reduce();
    }

    // Арифметические операторы
    Rational operator-() const { return Rational(-num, den); }
    Rational operator+() const { return *this; }

    Rational& operator+=(const Rational& rhs) {
        num = num * rhs.den + rhs.num * den;
        den *= rhs.den;
        reduce();
        return *this;
    }
    Rational& operator-=(const Rational& rhs) {
        return *this += -rhs;
    }
    Rational& operator*=(const Rational& rhs) {
        num *= rhs.num;
        den *= rhs.den;
        reduce();
        return *this;
    }
    Rational& operator/=(const Rational& rhs) {
        if (rhs.num == 0) throw std::domain_error("Division by zero");
        num *= rhs.den;
        den *= rhs.num;
        reduce();
        return *this;
    }

    // Сравнения
    bool operator==(const Rational& rhs) const {
        return num == rhs.num && den == rhs.den;
    }
    bool operator!=(const Rational& rhs) const { return !(*this == rhs); }
    bool operator<(const Rational& rhs) const {
        return num * rhs.den < rhs.num * den;
    }
    bool operator<=(const Rational& rhs) const { return !(rhs < *this); }
    bool operator>(const Rational& rhs) const { return rhs < *this; }
    bool operator>=(const Rational& rhs) const { return !(*this < rhs); }

    // Ввод/вывод
    friend std::ostream& operator<<(std::ostream& os, const Rational& r) {
        os << r.num;
        if (r.den != 1) os << '/' << r.den;
        return os;
    }
    friend std::istream& operator>>(std::istream& is, Rational& r) {
        int n, d = 1;
        char slash;
        is >> n;
        if (is.peek() == '/') {
            is >> slash >> d;
        }
        if (is) r = Rational(n, d);
        return is;
    }

    // Приведение к double
    explicit operator double() const { return static_cast<double>(num) / den; }
};

// Бинарные операторы через свободные функции
Rational operator+(Rational lhs, const Rational& rhs) { return lhs += rhs; }
Rational operator-(Rational lhs, const Rational& rhs) { return lhs -= rhs; }
Rational operator*(Rational lhs, const Rational& rhs) { return lhs *= rhs; }
Rational operator/(Rational lhs, const Rational& rhs) { return lhs /= rhs; }

int main() {
    Rational r1(1, 2), r2(1, 3);
    std::cout << r1 << " + " << r2 << " = " << r1 + r2 << "\n";
    std::cout << r1 << " - " << r2 << " = " << r1 - r2 << "\n";
    std::cout << r1 << " * " << r2 << " = " << r1 * r2 << "\n";
    std::cout << r1 << " / " << r2 << " = " << r1 / r2 << "\n";

    std::cout << std::boolalpha;
    std::cout << r1 << " < " << r2 << "? " << (r1 < r2) << "\n";

    double d = static_cast<double>(r1);
    std::cout << "r1 as double: " << d << "\n";

    Rational r3;
    std::cout << "Enter rational (format: a/b or a): ";
    std::cin >> r3;
    std::cout << "You entered: " << r3 << "\n";
}
```

Этот пример демонстрирует:
- Сокращение дробей при создании и после операций.
- Перегрузку унарных, бинарных, составных операторов, сравнений, ввода/вывода.
- Использование свободных функций для бинарных операторов.
- Приведение к `double`.
