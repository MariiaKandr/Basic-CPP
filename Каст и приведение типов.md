В языке C++ есть **явное** и **неявное** приведение. 
Неявное выполняется автоматически, но может быть опасным. 
Явное — с помощью специальных операторов, которые делают код понятнее и безопаснее.

### 1. static_cast — для «безопасных» преобразований на этапе компиляции
- Числовые типы (`int → double`, `double → int`).
- Преобразования в иерархии наследования, когда вы **точно знаете** тип.
- Преобразование `void*` к нужному типу.

```cpp
double pi = 3.1415;
int intPi = static_cast<int>(pi);  // 3, дробная часть отброшена

class Base {};
class Derived : public Base {};
Base* b = new Derived();
Derived* d = static_cast<Derived*>(b); // компилируется, но если b на самом деле не Derived — неопределённое поведение
```

### 2. dynamic_cast — безопасное приведение в иерархии наследования (RTTI)
- Работает только с полиморфными классами (есть хотя бы одна виртуальная функция).
- Проверяет реальный тип объекта во время выполнения.
- Для указателей: возвращает `nullptr` при ошибке.
- Для ссылок: бросает исключение `std::bad_cast`.

```cpp
class Animal { virtual ~Animal(){} };
class Dog : public Animal { void bark() {} };
Animal* a = new Dog();
Dog* d = dynamic_cast<Dog*>(a);
if (d) d->bark();   // безопасно
```

### 3. const_cast — добавляет или убирает `const` / `volatile`
- Единственный оператор, который может снять константность.
- Используйте **только** когда вы уверены, что объект не является изначально константным, или для вызова старых C-функций, которые не используют `const`.

```cpp
void oldFunc(char* s) { s[0] = 'A'; }
const char* msg = "hello";
// oldFunc(msg); // ошибка
char* tmp = const_cast<char*>(msg); // снимаем const — опасно, если строка в read-only памяти!
```

### 4. reinterpret_cast — низкоуровневое переинтерпретирование битов
- Самый опасный. Превращает указатель на один тип в указатель на другой без проверок.
- Используется только в системном программировании (работа с памятью, аппаратурой, бинарными данными).

```cpp
int x = 0x12345678;
char* bytes = reinterpret_cast<char*>(&x); // побайтовый доступ
uintptr_t addr = reinterpret_cast<uintptr_t>(&x); // адрес как целое
```

### 5. RTTI — определение типа во время выполнения
- `typeid(объект).name()` — возвращает имя типа.
- `typeid` требует полиморфного класса, если применяется к ссылке/указателю на базовый класс.
- `type_info` позволяет сравнивать типы.

```cpp
Base* b = new Derived();
if (typeid(*b) == typeid(Derived)) {
    cout << "Это Derived!" << endl;
}
```

---

### Когда используем?

1. **static_cast** — для числовых преобразований и приведения в иерархии, когда вы знаете реальный тип.
2. **dynamic_cast** — когда тип неизвестен и нужно безопасно преобразовать указатель базового класса в производный.
3. **const_cast** — для работы с константностью, особенно в старом коде.
4. **reinterpret_cast** — для побайтового доступа и сериализации.
5. **typeid** — для идентификации типов объектов.
