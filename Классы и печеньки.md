# Всё про классы и конструкторы на примере печенек 🍪🍪🍪

Представь, что мы печём печеньки. Запомни: **класс — это форма для печенья**. По одной форме можно напечь много печенек (объектов). У каждой печеньки есть свои свойства (сахар, мука, начинка) и действия (выпечь, съесть).

---

## 1. Класс — форма для печенья

```cpp
class Cookie {
public:
    int sugar;       // сколько грамм сахара
    void bake() {    // метод "выпечь"
        cout << "Печенька с " << sugar << " г сахара готова!";
    }
};
```

Создаём печеньки:
```cpp
Cookie myCookie;   // взяли форму и сделали печеньку
myCookie.sugar = 10;
myCookie.bake();   // "Печенька с 10 г сахара готова!"
```

## 2. Конструкторы — как замесить тесто и сразу добавить начинку

Конструктор — это **рецепт**, который запускается автоматически, когда ты создаёшь печеньку. Он нужен, чтобы задать начальные ингредиенты.

### 2.1 Конструктор по умолчанию — обычная печенька без ничего

```cpp
Cookie() : sugar(0) {
    cout << "Испеклась пустая печенька" << endl;
}
```

Вызывается, когда пишешь `Cookie a;`. Если не написать свой, компилятор создаст пустой (как форму без теста), но лучше задать значения самому.

### 2.2 Параметризованный конструктор — печенька с заданным количеством сахара

```cpp
Cookie(int s) : sugar(s) {}
```

Теперь можно: `Cookie sweet(20);` — сахара 20 г.

### 2.3 Делегирующий конструктор — один рецепт вызывает другой, чтобы не повторяться

Например, у нас есть двойная печенька (значение double), но мы хотим округлить до целого сахара и приготовить обычную:

```cpp
Cookie(double d) : Cookie(static_cast<int>(round(d))) {}
```

Сначала округлили 12.7 → 13, потом вызвали `Cookie(13)`. Удобно!

### 2.4 Конструктор копирования — копируем печеньку, но с новой формой

Представь: у тебя есть печенька с шоколадной начинкой. Ты хочешь сделать **точно такую же**, но отдельную, чтобы одну съесть, а другую оставить. Просто присвоить нельзя (будет та же начинка — два указателя на один кусок шоколада, и если удалить одну печеньку, вторая останется с дырой). Нужно **глубокое копирование** — сделать новую начинку точно такого же состава.

```cpp
Cookie(const Cookie& other) : sugar(other.sugar) {
    // если у other есть начинка (next), скопируем её рекурсивно
}
```

В задаче с узлами списка: каждый узел указывает на следующий. При копировании узла надо скопировать и весь хвост.

### 2.5 Конструктор перемещения — переложить начинку из одной печеньки в другую, а старую выбросить

Когда у тебя есть временная печенька (например, только что из печи), ты можешь **украсть** её начинку, чтобы не делать новую. Старой печеньке оставляем пустую форму.

```cpp
Cookie(Cookie&& other) noexcept : sugar(other.sugar), filling(move(other.filling)) {
    other.filling = nullptr;   // обнуляем чужую начинку
}
```

`noexcept` — обещание, что ничего не упадёт.

## 3. Деструктор — что происходит, когда печеньку съели

Когда печенька уничтожается (выход из кухни, или её удалили), вызывается деструктор. Можно вывести сообщение.

```cpp
~Cookie() {
    cout << "Печенька с " << sugar << " г сахара съедена :(" << endl;
}
```

---

## 4. Умные указатели — поварята, которые сами убирают за собой

### `std::unique_ptr` — эксклюзивный поварёнок

Это указатель, который **один владеет печенькой**. Нельзя скопировать — только переместить. Когда поварёнок уходит, он автоматически удаляет печеньку. Идеально для списка: каждый узел уникально владеет следующим.

```cpp
unique_ptr<Cookie> p1 = make_unique<Cookie>(10);  // поварёнок p1 держит печеньку
unique_ptr<Cookie> p2 = move(p1);                // теперь p2 владеет, p1 — пустой
```

В конструкторе копирования `Node` мы не можем скопировать `unique_ptr`, поэтому создаём новый узел через `make_unique<Node>(*other.next)`.

### `std::shared_ptr` — общий поварёнок (несколько поварят делят одну печеньку)

Можно копировать, счётчик ссылок. Пока хотя бы один поварёнок держит печеньку — она жива.

## 5. Приведение типов — превращаем одну печеньку в другую

### `dynamic_cast` — проверка, с чем мы имеем дело

У нас есть базовая форма `Cookie` (мука, сахар) и специальные: `ChocolateCookie`, `GingerCookie`. Если у нас указатель на `Cookie`, но мы подозреваем, что на самом деле это шоколадная, используем `dynamic_cast`. Он проверит и вернёт `nullptr`, если ошибся.

```cpp
ChocolateCookie* ch = dynamic_cast<ChocolateCookie*>(cookiePtr);
if (ch) { cout << "О, шоколадная!"; }
```

### `static_cast` — принудительное превращение (без проверки)

Скажем, я точно знаю, что это шоколадная, но компилятор не верит. Превращаю насильно. Опасно, если ошибёшься — печенька рассыплется.

```cpp
ChocolateCookie* ch = static_cast<ChocolateCookie*>(cookiePtr);
```

### `reinterpret_cast` — совсем дикое: прочитать печеньку как набор байтов

Представь, что ты берёшь печеньку и говоришь: «А теперь это кусок металла». Без проверок, просто переинтерпретируешь биты. Используется только в очень низкоуровневых задачах.

## 6. Наследование — когда одна печенька расширяет другую

```cpp
class Cookie {
public:
    virtual string taste() const { return "мучная"; }
    virtual ~Cookie() = default;   // виртуальный деструктор — обязательно!
};

class ChocolateCookie : public Cookie {
public:
    string taste() const override { return "шоколадная"; }
    void melt() const {}   // у шоколадной есть метод плавиться
};
```

`virtual` позволяет вызывать правильный метод даже через указатель на базовый класс. Деструктор базового класса тоже должен быть виртуальным, чтобы при удалении через указатель на `Cookie` правильно вызвался деструктор `ChocolateCookie`.

## 7. Глубокое копирование для динамического массива

Представь, что печенька содержит внутри **список начинок** (динамический массив). При копировании нужно не просто скопировать указатель на список, а создать новый список и скопировать каждую начинку.

```cpp
class Cookie {
    int* fillings;   // указатель на массив начинок
    size_t count;
public:
    // Конструктор копирования: глубокая копия
    Cookie(const Cookie& other) : count(other.count) {
        fillings = new int[count];
        copy(other.fillings, other.fillings + count, fillings);
    }

    // Конструктор перемещения: украли массив
    Cookie(Cookie&& other) noexcept : fillings(other.fillings), count(other.count) {
        other.fillings = nullptr;
        other.count = 0;
    }

    // Деструктор: убрали за собой
    ~Cookie() { delete[] fillings; }
};
```

## 8. Что делать, когда решаешь задачи

1. **Прочитай задание.** Выдели, какие конструкторы нужны (по умолчанию, параметризованный, копирования, перемещения, делегирующий) и что должен делать деструктор.
2. **Подключи библиотеки:** `<iostream>`, `<memory>`, `<string>`, `<cmath>` (для `round`), `<algorithm>` (для `copy`, `fill`).
3. **Напиши класс.** Сначала поля, потом конструкторы, методы в том порядке, как в задании.
4. **Конструктор копирования для списка** — не забывай рекурсивно копировать `next`: `next(other.next ? make_unique<Node>(*other.next) : nullptr)`.
5. **Конструктор перемещения** — забери `move(other.next)`, обнули чужой `next`, но `value` не трогай, если тест ожидает, что деструктор исходного выведет старое значение.
6. **Не забудь про методы** — как для работы с приватными полями, так и для поведения объекта.
7. **Деструктор** — только выведи сообщение, если просят. Умный указатель сам очистит память.
8. **Если тесты лезут в приватные поля** — сделай поля `public`. 
9. **При наследовании** — не забудь `virtual` деструктор в базовом классе.
10. **Приведения типов** — `dynamic_cast` для проверок, `static_cast` когда уверен, `reinterpret_cast` только когда точно нужно.

## 9. Пример задачи про Node (печенька с начинкой, которая указывает на следующую печеньку)

```cpp
#include <iostream>
#include <memory>
#include <cmath>
#include <string>
using namespace std;

class Node {
public:
    int value;                     // количество сахара в печеньке
    unique_ptr<Node> next;         // указатель на следующую печеньку (эксклюзивное владение)

    Node() : value(0), next(nullptr) {}
    Node(int val) : value(val), next(nullptr) {}
    Node(double val) : Node(static_cast<int>(round(val))) {}

    Node(const Node& other)
        : value(other.value),
          next(other.next ? make_unique<Node>(*other.next) : nullptr) {}

    Node(Node&& other) noexcept
        : value(other.value), next(move(other.next)) {
        other.next = nullptr;   // оставляем чужую печеньку без хвоста
        // значение не обнуляем — деструктор other выведет исходное число
    }

    ~Node() {
        cout << "~Node(" << value << ")" << endl;
    }
};
```
