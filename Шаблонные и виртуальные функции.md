## Часть 1. Шаблоны функций (Templates)

### Зачем нужны шаблоны?

Представьте, что мы написали функцию для нахождения максимального из двух целых чисел:

```cpp
int max(int a, int b) {
    return (a > b) ? a : b;
}
```

Теперь нам понадобилась такая же функция для чисел с плавающей точкой:

```cpp
double max(double a, double b) {
    return (a > b) ? a : b;
}
```

И для символов:

```cpp
char max(char a, char b) {
    return (a > b) ? a : b;
}
```

Вы видите, что тело функции абсолютно одинаковое, меняются только типы данных. Если бы мы писали код для каждого типа отдельно, это было бы утомительно и приводило бы к ошибкам (вдруг в одной из версий мы допустим опечатку?). Хотелось бы написать **одну** функцию, которая умеет работать с любым типом.

Именно для этого в C++ существуют **шаблоны** (templates). Шаблон функции — это не готовая функция, а **инструкция для компилятора**, как создавать функции для разных типов. Когда мы вызываем шаблонную функцию с конкретным типом, компилятор автоматически генерирует нужную версию. Это называется **инстанцированием шаблона**.

### Первый шаблон: максимум из двух значений

Давайте напишем шаблонную функцию `maxOfTwo`. Синтаксис шаблона начинается с ключевого слова `template`, после которого в угловых скобках `<>` перечисляются **параметры шаблона**. Чаще всего это параметры-типы, которые объявляются с помощью `typename` (или `class` — для параметров-типов это одно и то же).

```cpp
#include <iostream>
#include <string>

// Объявляем шаблон с одним параметром-типом T
template <typename T>
T maxOfTwo(T a, T b) {
    // Эта строчка будет работать для любого типа T, который поддерживает оператор >
    return (a > b) ? a : b;
}

int main() {
    // Компилятор видит вызов maxOfTwo(10, 20), понимает, что аргументы типа int,
    // и автоматически генерирует функцию int maxOfTwo(int, int)
    std::cout << "maxOfTwo(10, 20) = " << maxOfTwo(10, 20) << std::endl;

    // А здесь генерируется double maxOfTwo(double, double)
    std::cout << "maxOfTwo(3.14, 2.71) = " << maxOfTwo(3.14, 2.71) << std::endl;

    // И даже для строк! std::string поддерживает оператор > (лексикографическое сравнение)
    std::string s1 = "apple";
    std::string s2 = "banana";
    std::cout << "maxOfTwo(apple, banana) = " << maxOfTwo(s1, s2) << std::endl;

    // Для символов тоже работает
    std::cout << "maxOfTwo('A', 'Z') = " << maxOfTwo('A', 'Z') << std::endl;

    return 0;
}
```

**Важное замечание:** шаблонная функция не компилируется до тех пор, пока не будет вызвана с конкретным типом. Компилятор проверяет корректность только тех типов, которые реально используются. Если тип не поддерживает операцию `>`, вы получите ошибку компиляции именно в месте вызова.

### Проблема: разные типы аргументов

Что будет, если мы попытаемся вызвать `maxOfTwo(10, 3.14)`? Первый аргумент — `int`, второй — `double`. Компилятор не может однозначно определить, чему равен `T`. Это приведет к ошибке:

```
error: no matching function for call to 'maxOfTwo(int, double)'
```

Есть несколько способов решить эту проблему.

#### Способ 1: Явное указание типа

Мы можем сами сказать компилятору, какой тип использовать:

```cpp
std::cout << maxOfTwo<double>(10, 3.14) << std::endl;
```

Теперь `T` = `double`, и оба аргумента будут приведены к `double`. Это работает, но не всегда удобно.

#### Способ 2: Шаблон с двумя разными параметрами типа

Мы можем разрешить функции принимать аргументы разных типов. Но тогда возникает вопрос: какой тип возвращать? В C++11 и новее можно использовать `auto` и `decltype` для автоматического вывода возвращаемого типа:

```cpp
template <typename T1, typename T2>
auto maxOfTwoDifferent(T1 a, T2 b) -> decltype(a > b ? a : b) {
    return (a > b) ? a : b;
}

int main() {
    std::cout << maxOfTwoDifferent(10, 3.14) << std::endl;  // Выведет 10 (тип int? double? зависит от правил)
    std::cout << maxOfTwoDifferent(3.14, 10) << std::endl;  // Выведет 10
    return 0;
}
```

Здесь `decltype(выражение)` — это конструкция, которая "достает" тип выражения. Выражение `a > b ? a : b` в случае `int` и `double` будет иметь тип `double` (так как `int` неявно приводится к `double` в тернарном операторе). В C++14 можно писать еще короче:

```cpp
template <typename T1, typename T2>
auto maxOfTwoDifferent(T1 a, T2 b) {
    return (a > b) ? a : b;
}
```

### Специализация шаблона: когда общее решение не подходит

Иногда для какого-то конкретного типа нужно написать особую реализацию. Например, наша функция `maxOfTwo` для C-строк (`const char*`) будет сравнивать не содержимое строк, а **указатели** (адреса в памяти), что почти всегда неправильно. Для правильного сравнения нужно использовать функцию `strcmp`.

Для этого существует **явная специализация шаблона**:

```cpp
#include <cstring>

// Общий шаблон (как раньше)
template <typename T>
T maxOfTwo(T a, T b) {
    return (a > b) ? a : b;
}

// Явная специализация для типа const char*
template <>
const char* maxOfTwo<const char*>(const char* a, const char* b) {
    // strcmp возвращает >0 если a > b, 0 если равны, <0 если a < b
    return (std::strcmp(a, b) > 0) ? a : b;
}

int main() {
    const char* str1 = "apple";
    const char* str2 = "banana";
    std::cout << maxOfTwo(str1, str2) << std::endl;  // Теперь выведет "banana"
    return 0;
}
```

Обратите внимание на синтаксис: после `template` идут пустые угловые скобки `<>`, а после имени функции в угловых скобках указан конкретный тип.

### Нетиповые параметры шаблона

Параметром шаблона может быть не только тип, но и **значение** (целое число, перечисление, указатель). Это называется **нетиповым параметром**. Чаще всего это используется для работы с массивами фиксированного размера.

```cpp
#include <iostream>

// Шаблон функции, которая принимает массив по ссылке и его размер как нетиповой параметр N
template <typename T, int N>
void printArray(T (&arr)[N]) {   // arr - ссылка на массив из N элементов типа T
    std::cout << "Размер массива: " << N << std::endl;
    std::cout << "Элементы: ";
    for (int i = 0; i < N; ++i) {
        std::cout << arr[i] << " ";
    }
    std::cout << std::endl;
}

int main() {
    int numbers[] = {1, 2, 3, 4, 5};
    printArray(numbers);   // T = int, N = 5 (выводится автоматически)

    double values[] = {1.1, 2.2, 3.3};
    printArray(values);    // T = double, N = 3

    return 0;
}
```

Здесь `T (&arr)[N]` означает "ссылка на массив из N элементов типа T". Компилятор автоматически выводит и `T`, и `N`. Это очень удобно, чтобы не передавать размер массива отдельным параметром.

### Шаблоны с переменным числом аргументов (Variadic Templates)

Начиная с C++11, шаблоны могут принимать произвольное количество параметров. Это позволяет писать функции, которые работают с любым количеством аргументов. Например, функция `print`, которая выводит все переданные значения через пробел.

Реализуется это с помощью **рекурсии**: мы определяем функцию, которая обрабатывает первый аргумент, а затем вызывает себя для оставшихся.

```cpp
#include <iostream>

// Базовый случай: когда аргументов больше нет, просто завершаем вывод
void print() {
    std::cout << std::endl;
}

// Шаблон с переменным числом аргументов
template <typename T, typename... Args>   // Args... - пакет параметров
void print(const T& first, const Args&... rest) {
    std::cout << first << " ";            // Выводим первый аргумент
    print(rest...);                       // Рекурсивно вызываем для остальных
}

int main() {
    print(1);                             // 1
    print(1, 2, 3);                       // 1 2 3
    print(1, 2.5, "hello", 'c');          // 1 2.5 hello c
    return 0;
}
```

Здесь `typename... Args` объявляет **пакет параметров**, а `Args... rest` — это **распаковка пакета**. Многоточие `...` называется **ellipsis** и используется в нескольких местах: при объявлении пакета, при его распаковке, и в C++17 еще в **fold expressions**.

В C++17 появились **fold expressions** — более простой способ работать с пакетами без рекурсии. Например, сумма всех аргументов:

```cpp
template <typename... Args>
auto sum(Args... args) {
    return (args + ...);   // Унарная правая свертка: (((args1 + args2) + args3) + ...)
}
```

Этот код разворачивается компилятором в цепочку сложений.

### Ограничение шаблонов: SFINAE и концепты 

Иногда мы хотим, чтобы шаблонная функция работала только с типами, которые обладают определенными свойствами (например, имеют метод `size()`). Раньше для этого использовали сложную технику под названием **SFINAE** (Substitution Failure Is Not An Error — "неудачная подстановка не является ошибкой"). Идея в том, что если подстановка типа в шаблон приводит к синтаксической ошибке, эта перегрузка просто исключается из рассмотрения, а не вызывает ошибку компиляции.

В современном C++20 для этого ввели **концепты** (concepts) — явное описание требований к типу. Например:

```cpp
#include <concepts>

// Концепт: тип T должен поддерживать сложение
template <typename T>
concept Addable = requires(T a, T b) {
    { a + b } -> std::same_as<T>;
};

// Функция принимает только типы, удовлетворяющие концепту Addable
template <Addable T>
T add(T a, T b) {
    return a + b;
}
```

Это делает код более понятным и сообщения об ошибках более дружелюбными. Поскольку вы только начинаете, детали SFINAE можно пока пропустить, но знайте, что такие механизмы существуют.

## Часть 2. Виртуальные функции и полиморфизм

### Проблема: работа с объектами через указатель на базовый класс

Рассмотрим иерархию классов животных. У нас есть базовый класс `Animal` с методом `speak()`, и производные классы `Dog` и `Cat`, которые переопределяют этот метод.

```cpp
#include <iostream>
#include <string>

class Animal {
protected:
    std::string name;
public:
    Animal(const std::string& n) : name(n) {}
    void speak() const {
        std::cout << name << " издает звук" << std::endl;
    }
};

class Dog : public Animal {
public:
    Dog(const std::string& n) : Animal(n) {}
    void speak() const {
        std::cout << name << " говорит: Гав!" << std::endl;
    }
};

class Cat : public Animal {
public:
    Cat(const std::string& n) : Animal(n) {}
    void speak() const {
        std::cout << name << " говорит: Мяу!" << std::endl;
    }
};
```

Теперь представим, что мы хотим хранить разных животных в одном массиве и заставить их подать голос. Для этого мы используем указатели на базовый класс `Animal*`:

```cpp
int main() {
    Dog dog("Бобик");
    Cat cat("Мурка");

    Animal* animals[] = { &dog, &cat };

    for (Animal* animal : animals) {
        animal->speak();   // Что вызовется?
    }
    return 0;
}
```

**Вывод программы:**
```
Бобик издает звук
Мурка издает звук
```

Почему так? Потому что в классе `Animal` метод `speak()` **не виртуальный**. Когда компилятор видит вызов `animal->speak()`, он смотрит на тип указателя (`Animal*`) и вызывает метод `Animal::speak()`, не обращая внимания на то, что на самом деле объект является `Dog` или `Cat`. Это называется **раннее связывание** (early binding) — решение о том, какую функцию вызвать, принимается на этапе компиляции.

### Виртуальные функции: позднее связывание

Чтобы решить эту проблему, мы должны объявить метод в базовом классе как **виртуальный** с помощью ключевого слова `virtual`. Тогда компилятор создаст специальную таблицу виртуальных функций (vtable), и вызов будет происходить через нее. Это называется **позднее связывание** (late binding) или **динамический полиморфизм**.

```cpp
class Animal {
    // ...
public:
    virtual void speak() const {
        std::cout << name << " издает звук" << std::endl;
    }
    virtual ~Animal() {}   // О виртуальном деструкторе поговорим ниже
};
```

Теперь в производных классах мы **переопределяем** виртуальную функцию. Рекомендуется использовать ключевое слово `override`, чтобы явно указать компилятору, что мы переопределяем виртуальную функцию базового класса. Это помогает избежать ошибок (например, если сигнатура функции в базовом классе изменилась, а мы забыли обновить производный класс, компилятор выдаст ошибку).

```cpp
class Dog : public Animal {
public:
    Dog(const std::string& n) : Animal(n) {}
    void speak() const override {   // override необязателен, но крайне желателен
        std::cout << name << " говорит: Гав!" << std::endl;
    }
};

class Cat : public Animal {
public:
    Cat(const std::string& n) : Animal(n) {}
    void speak() const override {
        std::cout << name << " говорит: Мяу!" << std::endl;
    }
};
```

Теперь при вызове `animal->speak()` программа во время выполнения посмотрит в vtable объекта и вызовет правильную версию. Вывод будет:

```
Бобик говорит: Гав!
Мурка говорит: Мяу!
```

### Как работает vtable (упрощенно)

Каждый класс, имеющий хотя бы одну виртуальную функцию, содержит скрытый указатель (обычно 8 байт на 64-битной системе) на таблицу виртуальных функций — **vtable**. Эта таблица статическая для каждого класса и содержит адреса виртуальных функций. Когда вызывается виртуальная функция через указатель на базовый класс, происходит косвенный вызов: сначала из объекта берется указатель на vtable, затем из vtable по фиксированному смещению берется адрес нужной функции, и она вызывается.

Именно из-за этого косвенного вызова виртуальные функции работают немного медленнее обычных, но эта разница в большинстве приложений незначительна.

### Виртуальный деструктор — ОБЯЗАТЕЛЕН!

Если у вас есть иерархия классов с виртуальными функциями, вы почти наверняка будете удалять объекты через указатель на базовый класс. Рассмотрим пример:

```cpp
class Base {
public:
    ~Base() { std::cout << "~Base()" << std::endl; }
};

class Derived : public Base {
private:
    int* data;
public:
    Derived() { data = new int[100]; }
    ~Derived() { delete[] data; std::cout << "~Derived()" << std::endl; }
};

int main() {
    Base* ptr = new Derived();
    delete ptr;   // Вызовется только ~Base() !!!
    return 0;
}
```

Вывод: `~Base()`. Деструктор `Derived` не вызвался, память `data` не освобождена — **утечка памяти**!

Чтобы этого избежать, деструктор базового класса должен быть **виртуальным**:

```cpp
class Base {
public:
    virtual ~Base() { std::cout << "~Base()" << std::endl; }
};
```

Теперь при `delete ptr` произойдет поиск деструктора через vtable, и сначала вызовется `~Derived()`, а затем `~Base()`. Память освободится корректно.

**Правило:** если класс предназначен для наследования и у него есть хотя бы одна виртуальная функция, его деструктор **обязан** быть виртуальным. Даже если виртуальных функций нет, но вы планируете удалять объекты через указатель на базу, деструктор тоже должен быть виртуальным. В сомнительных случаях лучше сделать деструктор виртуальным — это не повредит.

### Абстрактные классы и чисто виртуальные функции

Иногда мы хотим определить в базовом классе только интерфейс (набор методов), но не предоставлять реализацию. Например, у всех фигур есть площадь, но формула для круга и прямоугольника разная. Мы не можем написать осмысленный код в базовом классе `Shape`.

Для этого служат **чисто виртуальные функции**. Они объявляются с помощью `= 0`:

```cpp
class Shape {
protected:
    std::string color;
public:
    Shape(const std::string& c) : color(c) {}
    virtual double area() const = 0;   // Чисто виртуальная функция
    virtual void draw() const = 0;
    virtual ~Shape() {}
};
```

Класс, содержащий хотя бы одну чисто виртуальную функцию, называется **абстрактным**. Нельзя создать объект абстрактного класса:

```cpp
Shape s("red");   // ОШИБКА! Shape - абстрактный класс
```

Но можно создавать объекты производных классов, которые **реализуют** все чисто виртуальные функции:

```cpp
class Circle : public Shape {
private:
    double radius;
public:
    Circle(const std::string& c, double r) : Shape(c), radius(r) {}
    double area() const override {
        return 3.14159 * radius * radius;
    }
    void draw() const override {
        std::cout << "Рисуем круг цвета " << color << std::endl;
    }
};
```

Абстрактные классы широко используются для создания **интерфейсов** (хотя в C++ нет отдельного ключевого слова `interface`, как в Java). Такой класс содержит только чисто виртуальные функции и, возможно, виртуальный деструктор.

### Множественное наследование

C++ позволяет классу наследоваться от нескольких базовых классов одновременно:

```cpp
class Printer {
public:
    void print(const std::string& text) {
        std::cout << "Печать: " << text << std::endl;
    }
};

class Scanner {
public:
    void scan() {
        std::cout << "Сканирование..." << std::endl;
    }
};

class MFU : public Printer, public Scanner {
    // МФУ наследует и печать, и сканирование
};

int main() {
    MFU device;
    device.print("Документ");
    device.scan();
    return 0;
}
```

Это удобно, но может привести к проблемам.

#### Проблема ромбовидного наследования (Diamond Problem)

Рассмотрим классический пример:

```cpp
class Animal {
public:
    std::string name;
    Animal(const std::string& n) : name(n) {}
};

class Mammal : public Animal {
public:
    Mammal(const std::string& n) : Animal(n) {}
};

class WingedAnimal : public Animal {
public:
    WingedAnimal(const std::string& n) : Animal(n) {}
};

class Bat : public Mammal, public WingedAnimal {
public:
    Bat(const std::string& n) : Mammal(n), WingedAnimal(n) {}
};
```

Иерархия образует ромб: `Bat` наследует от `Mammal` и `WingedAnimal`, а те оба наследуют от `Animal`. В результате объект `Bat` содержит **два** подобъекта `Animal`: один от `Mammal`, другой от `WingedAnimal`. Это приводит к неоднозначности:

```cpp
Bat bat("Бэтси");
// bat.name = "Batty"; // Ошибка! Какой из двух name?
std::cout << bat.Mammal::name << std::endl;   // Придется указывать путь явно
std::cout << bat.WingedAnimal::name << std::endl;
```

Более того, конструктор `Bat` должен инициализировать оба подобъекта `Animal`, что нелогично (летучая мышь — одно животное).

#### Решение: виртуальное наследование

Чтобы в объекте был **только один** экземпляр базового класса, нужно использовать **виртуальное наследование**:

```cpp
class Mammal : virtual public Animal {
    // ...
};

class WingedAnimal : virtual public Animal {
    // ...
};

class Bat : public Mammal, public WingedAnimal {
public:
    // Теперь конструктор Bat должен явно инициализировать Animal
    Bat(const std::string& n) : Animal(n), Mammal(n), WingedAnimal(n) {}
};
```

Теперь `Bat` содержит только один подобъект `Animal`, и доступ к `name` становится однозначным.

Виртуальное наследование добавляет небольшие накладные расходы (дополнительный указатель на виртуальную базу), поэтому его используют только тогда, когда это действительно необходимо. В современном C++ множественное наследование реализации (классов с данными) применяется редко. Гораздо чаще используют множественное наследование **интерфейсов** (абстрактных классов без данных), которое не вызывает проблемы ромба.

#### Задача 3 (Шаблоны): Шаблон с несколькими параметрами

Напишем функцию `convert`, которая преобразует значение из типа `Source` в тип `Target`.

```cpp
#include <iostream>
#include <string>

// Основной шаблон: использует static_cast
template <typename Target, typename Source>
Target convert(Source src) {
    return static_cast<Target>(src);
}

// Специализация: std::string -> int
template <>
int convert<int, std::string>(std::string src) {
    return std::stoi(src);
}

// Специализация: int -> std::string
template <>
std::string convert<std::string, int>(int src) {
    return std::to_string(src);
}

int main() {
    double d = convert<double>(10);           // int -> double
    int i = convert<int>(3.99);               // double -> int (отбрасывает дробную часть)
    int num = convert<int>(std::string("42")); // string -> int
    std::string str = convert<std::string>(100); // int -> string

    std::cout << d << " " << i << " " << num << " " << str << std::endl;
    return 0;
}
```

#### Задача 1 (Виртуальные функции): Базовая иерархия

Реализуем иерархию фигур с виртуальными функциями `area()` и `perimeter()`.

```cpp
#include <iostream>
#include <vector>
#include <cmath>

class Shape {
protected:
    std::string color;
public:
    Shape(const std::string& c) : color(c) {}
    virtual double area() const = 0;
    virtual double perimeter() const = 0;
    virtual void print() const {
        std::cout << "Фигура цвета " << color;
    }
    virtual ~Shape() {}
};

class Circle : public Shape {
    double radius;
public:
    Circle(const std::string& c, double r) : Shape(c), radius(r) {}
    double area() const override {
        return M_PI * radius * radius;
    }
    double perimeter() const override {
        return 2 * M_PI * radius;
    }
    void print() const override {
        Shape::print();
        std::cout << ", круг, радиус " << radius
                  << ", площадь " << area()
                  << ", периметр " << perimeter() << std::endl;
    }
};

class Rectangle : public Shape {
    double width, height;
public:
    Rectangle(const std::string& c, double w, double h) : Shape(c), width(w), height(h) {}
    double area() const override {
        return width * height;
    }
    double perimeter() const override {
        return 2 * (width + height);
    }
    void print() const override {
        Shape::print();
        std::cout << ", прямоугольник " << width << "x" << height
                  << ", площадь " << area()
                  << ", периметр " << perimeter() << std::endl;
    }
};

int main() {
    std::vector<Shape*> shapes;
    shapes.push_back(new Circle("красный", 5.0));
    shapes.push_back(new Rectangle("синий", 4.0, 6.0));

    for (Shape* s : shapes) {
        s->print();
        delete s;   // Благодаря виртуальному деструктору все корректно удалится
    }
    return 0;
}
```

## Заключение

Мы рассмотрели два фундаментальных механизма C++: **шаблоны** (статический полиморфизм) и **виртуальные функции** (динамический полиморфизм). Шаблоны позволяют писать обобщенный код, который компилятор адаптирует под конкретные типы, обеспечивая максимальную производительность. Виртуальные функции дают гибкость во время выполнения, позволяя работать с объектами разных классов через единый интерфейс.
